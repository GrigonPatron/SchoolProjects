### Part 5. Этап деплоя

**== Задание ==**

##### Подними вторую виртуальную машину *Ubuntu Server 22.04 LTS*.

Склонировал виртуальную машину с Ubuntu и назвал Ubuntu Server 2, также настроил каждой ВМ второй адаптер на внутреннюю сеть.

![1754672582169](images/readme/1754672582169.png)

Напиши этап для **CD**, который «разворачивает» проект на другой виртуальной машине.

##### Запусти этот этап вручную при условии, что все предыдущие этапы прошли успешно.

##### Напиши bash-скрипт, который при помощи **ssh** и **scp** копирует файлы, полученные после сборки (артефакты), в директорию */usr/local/bin* второй виртуальной машины.

*Тут тебе могут помочь знания, полученные в проекте DO2_LinuxNetwork.*

- Будь готов объяснить по скрипту, как происходит перенос.

Для подключения к виртуальным машинам использовал Remote-SSH из VSCode.

Далее, с помощью нового созданного файла "01-netcfg.yml" прописал конфигурацию netplan, соединил ВМ через routes и заранее прописал статические ip-адреса:

![1754681921557](images/readme/1754681921557.png)

Также, пропинговал машины:

![1754681984913](images/readme/1754681984913.png)

Итак, две машины "видят" друг друга и необходимо предоставить ключ доступа, чтобы в дальнейшем необходимые output файлы отправлялись на этапе деплоя с первой машины на вторую. Для этого сгенерировал ssh ключ. Сделал так:

- sudo su gitlab-runner для входа в права root
- ssh-keygen -t rsa -b 4096 -f ~/.ssh/id_rsa -N "" для создания ключа
- ssh-copy-id shadrict@192.168.100.44 для копирования ключа
- ssh shadrict@192.168.100.44 для проверки/подключения первой на вторую машину без ввода пароля

Изначально, после первого успешного подключения создаю маленький bash скрипт sentodeploy.sh

![1755276496312](images/readme/1755276496312.png)

Данный скрипт копирует скомпилированный cat и grep на вторую витртуальную машину, подключается и отправляет файлы в папку /usr/local/bin. Это работает, но не безопасно оставлять пароль в исходных файлах, поэтому через visudo разрешил на второй машине заходить под моим ником по SSH без пароля и переписал скрипт sentodeploy.sh в рамках школьного проекта допускаю, что так можно делать (в реальной работе можно сделать лучше, проблема в ограничениях школьного GitLab).
